## Prompt 1：解释 LoRA

输入：

```text
请用两句话解释 LoRA 的作用。
```

| 模型 | 输出摘要 |
| --- | --- |
| 基础模型 | 反复复读输入，并夹杂 `.linkedin` 片段。 |
| 已有 SFT | 能解释 LoRA 是低秩矩阵微调方法，并提到减少资源消耗。 |
| LoRA（仅助手回答损失） | 能给出更简短的 LoRA 解释，表达基本正确。 |
| QLoRA（仅助手回答损失） | 在该提示上仍出现复读，说明 40 步短训练不足以稳定改善生成。 |
| DPO | 输出接近已有 SFT，能正常解释 LoRA。 |

## Prompt 3：温和鼓励

```text

```

| 模型 | 输出摘要 |

## 小结

- 已有 SFT checkpoint 对通用中文问答的改善最明显。
- LoRA 仅助手回答损失版本在 LoRA 概念解释上有改善，但 120 步仍不足以稳定处理结构化输出。
- QLoRA 路线已经跑通，但短训练下生成质量波动较大，更适合先作为显存和流程验证。
- DPO 这轮使用的是小规模构造偏好数据，能验证链路，但不应解读为稳定偏好对齐效果。
